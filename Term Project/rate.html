Frameworks Experience Ratings


Did it perform to your expectations?
As it was our first time using Materialize or any kind of frontend framework, our expectations were quite high and Materialize definitely delivered the expectations. No, we wanted to create a public page that's beautiful but even though I tried to follow many tutorials to deploys / publicize the project, it was difficult.
Was it difficult to install or configure?
The installation and configuration of Materialize was not difficult. We did not have to go through that many steps to set it up. The installation and configuration of Django was more difficult because there are more prerequisites steps to go through before we can successfully installed the Django itself.

Was it easy to create the page?
Materialize was very easy to use and the in-built functions made it very simple to create an attractive page with not a lot of code at all. Django is designed to be front-end compatible, so it will do nothing to get in the way of you adding whatever you like to the front-end, so it wasn't very difficult to create the html page. However, it was not very easy to use as a newbie.


Was the learning curve steep compared with HTML/CSS/Javascript/PHP?
Materialize's learning curve is almost the same as HTML/CSS because both frameworks are very similar to one another. The main differences we found between them were only their components and the grid system. The learning curve is a bit steeper than the other frameworks because there isn't many straight forward tutorials to help in aid with the development / deployment of Django projects. It is a great tool in building databases for blogs and more complicated websites that require a DB. You can do so with one click.
Would you have done things differently in retrospect?
If we had a little more time we would have tried to display how the buttons work in the demo page and make the calendar grid more responsive. As for Django, we would have picked a less complicated back-end framework.
Did you regret your choice of framework?
No we do not have any regrets because we were able to make a very attactive and functional page which was not complicated at all to create. We did not regret choosing Django despite the difficulty in finding proper tutorials. It wasn't that difficult of a framework to work with. We just needed more time to get familiar with the framework. I think it is a very powerful framework.
